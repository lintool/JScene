JScene

A proof-of-concept in-browser JavaScript-based search engine

JavaScript engines inside modern web browsers are capable of running sophisticated multi-player games, rendering impressive 3D scenes, and supporting complex, interactive visualizations. Can this processing power be harnessed for information retrieval?

JScene (pronounced Jay-Scene, rhymes with Lucene) explores the feasibility of building a JavaScript search engine that runs completely self-contained on the client side within the browser—this includes building the inverted index, gathering terms statistics for scoring, and performing query evaluation. The design takes advantage of the IndexDB API, which is implemented by the LevelDB key-value store inside Google's Chrome browser.

Experiments show that although the performance of the JavaScript prototype falls far short of the open-source Lucene search engine, it is sufficiently responsive for interactive applications. This feasibility demonstration opens the door to interesting applications and architectures.

For more details, see:

  • Jimmy Lin. Building a Self-Contained Search Engine in the Browser. Proceedings of the ACM SIGIR International Conference on the Theory of Information Retrieval (ICTIR 2015), September 2015, Northampton, Massachusetts.
  • Jimmy Lin. On the Feasibility and Implications of Self-Contained Search Engines in the Browser. arXiv:1410.4500, October 2014. (Longer version of above paper.)
